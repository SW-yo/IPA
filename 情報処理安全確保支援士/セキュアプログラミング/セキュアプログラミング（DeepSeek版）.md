### 💡 令和8年度春期 予想問題例
今後のCBT化の流れを踏まえ、**プログラムの読解力と実務的なデータ処理能力**に焦点を当てた予想問題を作成しました。

**【設問】** 以下のプログラムは、オンラインショップの「注文処理システム」の一部を擬似言語で記述したものである。注文リストから特定の条件を満たす商品を抽出し、総額を計算する処理を行っている。プログラム中の空欄(a)～(c)を補え。

**【プログラムの説明】**
*   変数 `orderList` は、全注文を格納した配列である。各要素は `{ productId: "A101", category: "electronics", price: 15000, status: "shipped" }` 形式のオブジェクト。
*   関数 `calculateTotal` は、引数で指定されたカテゴリとステータスに一致する注文の合計金額を返す。
*   プログラム内では、高階関数 `filter` と `reduce` が使用されている。

**【プログラム (ECMAScript 風擬似言語)】**
```javascript
function calculateTotal(orderList, targetCategory, targetStatus) {
    // (a) 条件に合致する注文のみを抽出する
    const filteredOrders = orderList.filter( (order) => {
        return (order.category === targetCategory) ① (order.status === targetStatus);
    });

    // (b) 抽出された注文の価格配列に変換する
    const prices = filteredOrders.map( (order) => ② );

    // (c) 価格の合計を計算する。初期値は0とする。
    const total = prices.reduce( (sum, price) => ③ , 0);
    return total;
}
```

**【解答例】**
(a) **`&&`** (論理積を表す演算子。両条件を満たす必要があるため)
(b) **`order.price`** (注文オブジェクトから価格プロパティを取得)
(c) **`sum + price`** (`reduce`メソッドで合計を累積計算)

> **予想出題の意図**：この問題は、CBT化後の傾向として想定される「実務的なデータ構造操作」と「高階関数を用いたロジック理解」を組み合わせています。ECMAScriptの重要な概念である`filter`、`map`、`reduce`のチェーンを、擬似言語を通じて正しく読み書きできるかが問われています。
